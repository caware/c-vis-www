Raw Data

The data used by the visualisation tool is directly available for download at http://www.cl.cam.ac.uk/meters/elec/.

The data is organised in a directory structure which mimics the electricity distribution tree. A given directory may have data files and/or subdirectories.

There are data files with 2 minute time series (one per day) and with hourly time series (one per month). Thus for example:

http://www.cl.cam.ac.uk/meters/elec/primary-cs1-riser/F-lighting/

is a directory that contains the reading for the lighting circuit on the First floor that comes off the CS1 riser. The data files in that directory are of the form

S-m23-yyyy-mm-dd.json (for the 2 minute time series)

S-m23-yyyy-mm,json (for the 1 hour time series.)

If one looks inside any of the data files once can discern that the data is concerned with lighting on the First floor in the North section of the building.

Data files are in JavaScript Object Notation {http://www.json.org/} (json). This is so that the files can be easily manipulated in JavaScript. Indeed our visualisation tools are built on top of JavaScript packages that do most of the heavy lifting. We will supply some very simple code to help you get started if you want to perform any analysis of the data. (Or you can always look at the source of the tool itself - after all you download it to run it...)

Each file contains a small number of descriptive tagged values and a substantial time series whose pairs are a timestamp and an average power in kilowatts for that interval. A JavaScript time stamp is epoch milliseconds since 1 Jan 1970.

More tags will be added in the future. Current tags include:

  • label: ??
  • path: the directory to which the data file belongs
  • description: an indication of what the electricity is used for, eg lighting, sockets, air conditioning...
  • room: where the meter is located
  • coverage: what physical area the electricity is used in
  • type: the type of meter from which the data was obtained
  • ts: ???

We frequently regenerate the published data from the \textit{real} raw data so new tags will be added to old time series, (and old ones may disappear - although at some point we will commit to backwards compatibility)
